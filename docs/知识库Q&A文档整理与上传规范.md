# 知识库 Q&A 文档整理与上传规范

**版本：V1.0**  
**适用范围：需要上传至企业知识库，并使用“问答（Q&A）”方式解析的 Word 文档**  
**推荐文件格式：`.docx`**

---

## 一、规范目的

本规范用于保证业务部门整理的 Word 问答文档能够被知识库正确解析，避免发生以下问题：

- 问题没有形成独立知识块；
- 多个问题被合并到同一个答案中；
- Word 中可以看到答案，但知识库解析结果为空或不完整；
- 图片与对应问题关联错误；
- 重复问题生成多个知识块；
- 文档标题被识别成没有答案的问题；
- 用户搜索时无法准确命中对应答案。

核心原则：**一个独立问题对应一个独立答案，并形成一个独立 Q&A 知识块。**

---

## 二、标准文档结构

推荐统一使用以下层级：

| 内容类型 | Word/WPS 样式 | 是否必须 |
|---|---|---|
| 文档名称 | “标题”样式，或普通正文加粗居中 | 必须 |
| 一级业务模块 | 标题 1（Heading 1） | 必须 |
| 具体问题 | 标题 2（Heading 2） | 必须 |
| 答案内容 | 正文（Normal） | 必须 |
| 答案补充说明 | 正文（Normal） | 必须 |
| 图片说明 | 正文（Normal） | 建议 |
| 问答分隔线 | 正文（Normal） | 可选 |

标准结构示例：

```text
人事常见问题解答                     ← 文档标题

一、考勤管理                         ← 标题 1

1.1 员工忘记打卡怎么办？             ← 标题 2

答：请在3天内提交补卡申请。           ← 正文

=======================================================

1.2 补卡是否影响全勤奖？             ← 标题 2

答：符合补卡条件并在规定时间内完成补卡，不影响全勤奖。
```

### 2.1 文档总标题

文档最上方的总标题，例如“人事常见问题解答”，建议使用 Word/WPS 的“标题”样式，或者使用普通正文加粗居中。

总标题不要设置为“标题 2”，否则可能被识别为一个没有答案的空问题。

### 2.2 业务模块标题

业务模块使用“标题 1”，例如：

```text
一、考勤管理
二、薪酬福利
三、离职管理
```

### 2.3 具体问题

所有具体问题统一使用“标题 2”。

不得仅通过以下方式把文字制作成“看起来像标题”：

- 普通正文加粗；
- 调大字号；
- 修改字体颜色；
- 使用格式刷但没有修改段落样式；
- 使用 `Normal` 或 `Normal Web` 样式；
- 将问题放入文本框、形状或图片中。

知识库读取的是 Word 内部保存的段落样式，不是文字的视觉效果。

### 2.4 答案

答案及其补充说明统一使用“正文（Normal）”样式。

问题和答案必须是两个独立段落。

正确示例：

```text
3.9 什么是销假？

答：销假是指取消已经审批通过的休假申请。
```

错误示例：

```text
3.9 什么是销假？答：销假是指取消已经审批通过的休假申请。
```

## 三、问题编写规范

### 3.1 每个问题必须有唯一编号

推荐使用分级编号：

```text
1.1
1.2
1.3
2.1
2.2
```

也可以使用连续编号：

```text
1、
2、
3、
```

同一份文档内必须保持编号格式一致。

### 3.2 不允许重复编号

错误示例：

```text
6.4 休息日加班如何申请？
6.4 加班到凌晨如何打卡？
```

正确示例：

```text
6.4 休息日加班如何申请？
6.5 加班到凌晨如何打卡？
```

### 3.3 不允许重复问题

同一个问题不能在文档中重复出现。重复问题会产生重复知识块，可能导致检索结果、引用或原文件重复。

### 3.4 问题必须表达完整

不推荐：

```text
工资条
离职
社保
附件要求
```

推荐：

```text
工资条可以在哪里查看？
员工离职需要提前多少天提出申请？
入职后什么时候开始购买社保？
报销单需要上传哪些附件？
```

### 3.5 问题建议以问号结尾

推荐统一使用中文问号 `？`。

### 3.6 一个问题只表达一个主要事项

错误示例：

```text
如何申请年假、工资什么时候发、社保如何转移？
```

应拆分成三个独立问题。

## 四、答案编写规范

### 4.1 每个问题必须有有效答案

不允许出现以下情况：

- 只有问题，没有答案；
- 答案为空；
- 答案只有“详见附件”；
- 答案只有一个无法访问的链接；
- 答案完全放在图片、批注、页眉、页脚或文本框中。

### 4.2 答案建议以“答：”开头

推荐：

```text
答：员工可通过企业微信工作台进入相关系统提交申请。
```

同一份文档中应统一使用“答：”或“回答：”，不要混用多种格式。

### 4.3 答案可以包含多个正文段落

一个问题之后、下一个“标题 2”问题之前的正文段落，都会被视为当前问题的答案。

```text
2.1 离职申请如何提交？

答：员工本人需要在 EHR 系统中发起离职流程。

操作路径：企业微信 → 工作台 → EHR系统 → 员工自助 → 我的申请。

试用期员工需提前3天申请，正式员工需提前30天申请。
```

以上答案段落全部使用“正文”样式。

### 4.4 答案内容应完整明确

根据业务需要，答案应尽量说明：

- 适用对象；
- 办理条件；
- 办理路径；
- 时间要求；
- 所需材料；
- 特殊情况；
- 联系部门。

### 4.5 答案中的列表不能设置成问题标题

答案中的步骤可以使用普通编号：

```text
答：申请时需要提供以下材料：

1. 身份证复印件；
2. 对应业务申请表；
3. 审批记录；
4. 其他相关证明材料。
```

这些编号段落必须使用“正文”样式，不能设置为“标题 2”。

## 五、图片整理规范

### 5.1 图片必须放在对应答案下方

```text
4.1 如何提交报销申请？

答：进入费控系统后，选择对应报销类型并填写申请。

操作界面如下：

[图片]
```

不要将所有图片集中放在文档最后。

### 5.2 图片不能代替全部文字答案

不推荐：

```text
8、所有附件上传要求：

[只有一张图片]
```

推荐：

```text
8、报销申请需要上传哪些附件？

答：报销申请需要根据业务类型上传发票、付款凭证、审批记录及其他证明材料，具体要求如下图所示。

[图片]
```

纯图片答案不利于文本检索，也可能导致用户只收到图片而没有明确文字说明。

### 5.3 图片要求

- 使用清晰的 PNG 或 JPG 图片；
- 删除无关页面区域；
- 对关键按钮使用箭头或红框标注；
- 图片前增加文字说明；
- 图片后建议增加图注；
- 图片应设置为“与文字在一行”的嵌入式图片。

尽量避免浮动图片、文本框图片、SmartArt、组合图形、页眉页脚图片和外部链接图片。

## 六、表格整理规范

表格可以使用，但不建议将问题和答案全部放入复杂表格。

推荐先提供文字结论，再附表格：

```text
7.1 各地区社保缴纳时间有什么区别？

答：不同地区的社保缴纳时间如下表所示。
```

表格要求：

- 必须有表头；
- 避免大量合并单元格；
- 一个单元格中不要放多个问题；
- 不要将表格制作成截图作为唯一答案；
- 表格中的重要结论应在正文中再说明一次；
- 复杂表格可能单独生成表格知识块，不一定与前面的问题完全绑定。

---

## 七、修订与批注处理规范

### 7.1 上传前必须接受所有修订

Word/WPS 开启“修订”后，页面上可以看到的文字可能保存在特殊修订节点中。未接受修订可能导致：

- Word 中能看到答案，知识库解析为空；
- 答案只解析出一部分；
- 新增问题没有进入知识库；
- 已删除内容仍保留在文档内部；
- 图片出现重复或关联异常。

上传前必须：

1. 进入“审阅”；
2. 选择“接受所有修订”；
3. 选择“停止修订”；
4. 删除不需要的批注；
5. 保存文件；
6. 关闭并重新打开文件确认。

“无标记”或“最终状态”只是不显示修订，不代表修订已经被接受。

### 7.2 重要内容必须放在正文中

重要内容不要只放在：

- 页眉或页脚；
- 批注；
- 文本框；
- 形状或 SmartArt；
- 脚注或尾注；
- 隐藏文字；
- 修订记录。

## 八、Word/WPS 样式确认方法

1. 打开“显示/隐藏编辑标记”，显示段落符号 `¶`；
2. 将光标放在问题段落内；
3. 选中整个问题段落，最好包含末尾的 `¶`；
4. 在样式区域选择“标题 2”；
5. 不要只选中文字后调整字体、字号或加粗；
6. 保存文件并关闭；
7. 重新打开文件；
8. 打开“导航窗格”或“文档结构”；
9. 确认所有问题均作为独立标题显示。

如果 WPS 界面显示“标题 2”，但导航窗格中没有该问题，可以采用以下方法：

1. 复制一个已经正确设置的完整问题段落；
2. 替换为当前问题文字；
3. 删除原来的错误段落；
4. 保存、关闭并重新打开确认。

## 九、空行、分隔线与分页

- 问题和答案之间可以有一个空行；
- 不建议使用大量连续空行；
- 问题和答案可以跨页；
- 不要使用大量空行实现分页，应使用正式分页符；
- 问答是否正确切分主要取决于段落样式，不取决于问题和答案之间的视觉距离；
- 分隔线必须使用正文样式。

推荐分隔线：

```text
=======================================================
```

## 十、链接规范

答案中包含系统入口或外部网址时，应同时写明系统名称、操作路径、链接用途和完整网址。

推荐：

```text
答：可登录国家医保服务平台办理异地就医备案。

网址：https://fuwu.nhsa.gov.cn/

操作路径：个人服务 → 异地就医 → 异地就医备案。
```

不推荐只写“点击这里”“详见链接”或“打开这个页面”。

## 十一、文件与版本管理

### 11.1 文件格式

必须使用 `.docx`，不建议使用：

- `.doc`；
- `.docm`；
- 带密码的 Word 文件；
- 无法正常打开的损坏文件；
- 仅由扫描图片组成的 Word 文件。

### 11.2 文件命名

推荐：

```text
部门名称-业务模块-Q&A-日期-版本.docx
```

例如：

```text
人事部-考勤管理-Q&A-20260721-V1.0.docx
财务部-费用报销-Q&A-20260721-V1.0.docx
```

不建议使用“新版”“最终版”“最终修改版2”“新建文档”等含义不明确的文件名。

### 11.3 版本管理

修改内容后必须更新版本号或日期：

```text
V1.0：首次发布
V1.1：修改部分问答
V2.0：大范围更新制度内容
```

同一知识库中不要同时保留多个内容基本相同或相互冲突的版本。

## 十二、上传前检查清单

### 12.1 文件状态

- [ ] 文件格式为 `.docx`
- [ ] 文件能够正常打开
- [ ] 文件没有密码
- [ ] 已接受所有修订
- [ ] 已停止修订
- [ ] 已删除无关批注
- [ ] 已关闭并重新打开，确认修改已经保存

### 12.2 问题检查

- [ ] 每个问题都有唯一编号
- [ ] 没有重复编号
- [ ] 没有重复问题
- [ ] 问题表达完整
- [ ] 问题建议以问号结尾
- [ ] 每个问题使用“标题 2”
- [ ] 导航窗格中能够看到所有问题
- [ ] 没有使用 `Normal Web` 作为问题样式

### 12.3 答案检查

- [ ] 每个问题都有答案
- [ ] 问题和答案分属不同段落
- [ ] 答案使用“正文”样式
- [ ] 答案建议以“答：”开头
- [ ] 操作路径、时间、条件和材料描述完整
- [ ] 答案内容没有放在批注或文本框中
- [ ] 答案中的普通编号没有被设置为问题标题

### 12.4 图片和表格检查

- [ ] 图片位于对应问题的答案下方
- [ ] 图片清晰可见
- [ ] 图片设置为嵌入式
- [ ] 图片前有必要的文字说明
- [ ] 没有完全依赖图片作为答案
- [ ] 复杂表格的重要结论已转换成文字
- [ ] 没有在一个表格单元格中放多个问题

## 十三、上传后的验收检查

### 13.1 数量检查

源文件中的独立问题数，应与知识库生成的有效 Q&A chunk 数基本一致。

例如：

```text
源文件有65个独立问题
知识库应生成65个有效问答 chunk
```

如果文档包含复杂表格，可能额外生成表格 chunk，需要单独确认。

### 13.2 空答案检查

检查是否存在：

- 只有问题，没有答案；
- 只有文档标题，没有答案；
- 问题内容正常，但答案为空；
- 答案只有图片，没有文字。

### 13.3 抽样检查

至少检查：

- 文档开头的问题；
- 文档中间的问题；
- 文档最后的问题；
- 带图片的问题；
- 答案较长的问题；
- 多段落答案；
- 操作路径类问题。

### 13.4 搜索验证

使用源文件中的问题原文进行搜索，确认：

- 能命中正确问题；
- 答案完整；
- 图片正确；
- 没有重复结果；
- 没有引用错误文件。

### 13.5 异常排查顺序

如果 chunk 数量与源问题数不一致，应依次检查：

1. 问题是否使用正确的标题样式；
2. 是否存在 `Normal Web` 问题段落；
3. 问题和答案是否写在同一个段落；
4. 是否存在未接受的修订；
5. 是否存在重复编号或重复问题；
6. 是否把重要内容放在图片、表格或文本框中；
7. 是否上传了尚未保存的旧版本文件。

## 十四、标准 Q&A 文档模板

```text
部门名称—业务模块常见问题解答

一、业务模块一

1.1 第一个问题？

答：第一个问题的完整答案。

=======================================================

1.2 第二个问题？

答：第二个问题的完整答案。

如有操作步骤：

1. 第一步；
2. 第二步；
3. 第三步。

操作界面如下：

[图片]

=======================================================

二、业务模块二

2.1 第三个问题？

答：第三个问题的完整答案。
```

对应样式：

```text
文档名称       → “标题”样式或正文加粗居中
一级业务模块   → 标题 1
具体问题       → 标题 2
答案           → 正文
分隔线         → 正文
图片说明       → 正文
```

---

## 十五、强制退回条件

存在以下任一情况时，知识库管理员可以退回文件：

- 未接受所有修订；
- 大量问题使用正文或 `Normal Web` 样式；
- 问题和答案写在同一个段落；
- 多个问题共用一个答案且没有明确说明；
- 存在重复编号或重复问题；
- 存在没有答案的问题；
- 答案全部依赖截图；
- 重要内容位于文本框、批注、页眉或页脚；
- 图片模糊，无法看清操作内容；
- 文件版本和生效日期不明确；
- 同一知识库同时存在多个相互冲突的版本。

---

## 十六、业务部门提交信息

提交知识库文档时，应同时提供：

- 文件所属部门；
- 文件负责人；
- 文件版本；
- 制度或内容生效日期；
- 预计问题数量；
- 是否包含图片；
- 是否包含表格；
- 是否已完成修订清理；
- 是否允许向员工发送原文件；
- 后续内容更新联系人。

提交后，由知识库管理员完成解析和抽样验收。验收通过后，文件方可正式用于知识库问答。
